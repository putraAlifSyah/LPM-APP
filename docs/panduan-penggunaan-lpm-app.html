01

Pendahuluan

Pengenalan umum tentang Sistem Informasi LPM dan peran masing-masing pengguna.

1.1 Tentang Aplikasi

SI-LPM (Sistem Informasi Lembaga Penjaminan Mutu) adalah aplikasi web yang dirancang khusus untuk mendukung seluruh proses penjaminan mutu internal di Stikes Datu Kamanre. Aplikasi ini menyediakan fitur-fitur utama:

Modul Fungsi
📊 Dashboard Ringkasan statistik dokumen, audit, capaian Monev, dan pengumuman terbaru
📄 Manajemen Dokumen Upload, kelola, versioning, dan pencarian dokumen mutu (SPMI, Kurikulum, VMTS, dll.)
🔍 Audit Internal (AMI) Perencanaan sesi audit, pencatatan temuan, tindak lanjut, dan verifikasi
📈 Monitoring & Evaluasi Indikator kinerja, input capaian, dan laporan visual dengan grafik
📰 Berita & Pengumuman Publikasi informasi dari LPM ke seluruh sivitas akademika
👥 Manajemen Pengguna Kelola akun pengguna, role, dan hak akses (khusus Admin)
👤 Profil Edit data diri dan ganti kata sandi

1.2 Role Pengguna

Sistem mengenal 4 peran (role) dengan hak akses yang berbeda:

Role Label di Sistem Deskripsi
ADMIN Administrator Akses penuh ke seluruh fitur, termasuk manajemen pengguna, buat berita, dan hapus dokumen
AUDITOR Auditor Fokus pada kegiatan audit: membuat sesi audit, mencatat dan memverifikasi temuan
DOSEN Dosen / Staff Upload dokumen mutu dan input data capaian Monitoring & Evaluasi
PIMPINAN Pimpinan Akses untuk melihat seluruh data (dashboard, laporan, dokumen) secara read-only

1.3 Tabel Akses Fitur per Role

Berikut adalah matriks hak akses untuk setiap aksi dalam sistem:

02

Login ke Sistem

Panduan langkah demi langkah untuk masuk ke aplikasi SI-LPM.

2.1 Cara Masuk

Untuk mengakses sistem, buka browser dan ketikkan alamat aplikasi yang telah ditentukan.

  1. Buka halaman login
    Ketikkan alamat aplikasi di browser (contoh: http://localhost:3000 untuk development). Anda akan langsung diarahkan ke halaman login.
  2. Masukkan Email
    Isi kolom Email dengan alamat email kampus Anda yang telah didaftarkan oleh Administrator.
    Contoh format: nama@stikesdk.ac.id
  3. Masukkan Password
    Isi kolom Password dengan kata sandi yang diberikan oleh Administrator. Password bersifat case-sensitive (huruf besar/kecil dibedakan).
  4. Klik tombol "Masuk"
    Sistem akan memverifikasi kredensial Anda. Jika berhasil, Anda akan diarahkan ke halaman Dashboard.
⚠️ Jika gagal login: Pesan error "Email atau password salah" akan muncul. Pastikan Anda memasukkan email dan password dengan benar. Jika tetap gagal, hubungi Administrator.

2.2 Lupa Password

Jika Anda lupa password, silakan hubungi Administrator untuk mereset password akun Anda. Administrator dapat membuat akun baru atau menghubungi tim teknis.

ℹ️ Keamanan Session Sesi login Anda berlaku selama 7 hari. Setelah itu, Anda perlu login kembali. Untuk keamanan, selalu logout jika menggunakan komputer bersama.

05

Manajemen Dokumen

Panduan lengkap untuk mengelola dokumen mutu: upload, pencarian, versioning, dan penghapusan.

5.1 Melihat Daftar Dokumen

Buka menu Dokumen di sidebar. Halaman ini menampilkan tabel seluruh dokumen dengan informasi:

Kolom Keterangan
DokumenJudul dokumen, nomor versi, dan nama pengunggah
KategoriBadge berwarna: SPMI (biru), Kurikulum (hijau), VMTS (ungu), Renstra (kuning), dll.
StandarKode standar BAN-PT (misal: C.1.1, C.6.2)
ProdiProgram studi terkait (jika spesifik ke prodi tertentu)
UkuranUkuran file (KB/MB)
TanggalTanggal upload dokumen
AksiTombol Detail (👁️) dan Download (⬇️)

5.2 Filter & Pencarian

Di bagian atas tabel, terdapat toolbar pencarian dan filter:

  • 🔍 Kolom Pencarian Ketik kata kunci untuk mencari berdasarkan judul, tag, atau standar BAN-PT
  • 📁 Filter Kategori Pilih dari dropdown: SPMI, Kurikulum, VMTS, Renstra, Laporan, Formulir, Lainnya
  • 📋 Filter Standar Filter berdasarkan kode standar BAN-PT yang tersedia
  • 🏫 Filter Prodi Filter dokumen berdasarkan program studi: S1 Keperawatan, D3 Kebidanan, S1 Farmasi, D3 Keperawatan, Profesi Ners
ℹ️ Info Di bagian bawah tabel ditampilkan jumlah dokumen yang cocok dari total keseluruhan, contoh: "Menampilkan 3 dari 6 dokumen".

5.3 Upload Dokumen Baru

⚠️ Hak Akses Tombol Upload hanya muncul untuk role: ADMIN AUDITOR DOSEN
  1. Klik tombol "Upload" di pojok kanan atas toolbar
    Modal dialog "Upload Dokumen Baru" akan terbuka.
  2. Isi formulir upload dengan data berikut:
Field Wajib Keterangan
Judul Dokumen Ya Nama/judul dokumen. Contoh: "Kebijakan SPMI 2024"
Kategori Ya Pilih dari: SPMI, Kurikulum, VMTS, Renstra, Laporan, Formulir, Lainnya
Prodi Tidak Pilih prodi jika dokumen spesifik ke prodi tertentu, atau "Tidak spesifik"
Standar BAN-PT Tidak Kode standar, contoh: C.1.1, C.6.2
Tags Tidak Kata kunci pencarian, pisahkan dengan koma. Contoh: spmi, kebijakan, mutu
Deskripsi Tidak Keterangan singkat tentang isi dokumen
File Dokumen Ya Pilih file dari komputer. Format: PDF, DOCX, XLSX, PPTX. Maksimal ~10MB
  1. Klik "Upload" untuk mengunggah dokumen. Sistem akan menyimpan file dan metadata.
  2. Dokumen berhasil — Modal akan tertutup dan daftar dokumen otomatis diperbarui.

5.4 Detail Dokumen

Klik ikon mata (👁️) atau judul dokumen untuk melihat halaman detail yang menampilkan:

  • 📋 Informasi LengkapJudul, deskripsi, kategori, standar BAN-PT, tags, prodi, versi, pengunggah, dan tanggal
  • ⬇️ DownloadTombol untuk mengunduh file dokumen
  • 📜 Riwayat VersiDaftar semua versi sebelumnya beserta catatan perubahan

5.5 Edit Metadata Dokumen

Hanya tersedia untuk role yang memiliki izin upload.

  1. Buka halaman detail dokumen
  2. Klik tombol "Edit"
  3. Ubah field yang diinginkan (judul, kategori, prodi, standar, tags, deskripsi)
  4. Klik "Simpan"

5.6 Upload Versi Baru

Untuk memperbarui file dokumen tanpa kehilangan versi lama:

  1. Buka halaman detail dokumen
  2. Klik tombol "Versi Baru"
  3. Pilih file baru dari komputer
  4. Isi Catatan Perubahan (opsional, contoh: "Revisi sesuai temuan audit")
  5. Klik "Upload Versi"
✅ Versi Otomatis File lama akan disimpan ke riwayat versi secara otomatis. Nomor versi akan bertambah (v1 → v2 → v3, dst.).

5.7 Hapus Dokumen

⛔ Hanya Admin Hanya pengguna dengan role ADMIN yang dapat menghapus dokumen.

Penghapusan bersifat soft delete — dokumen tidak benar-benar dihapus dari database, melainkan ditandai sebagai "dihapus" dan tidak lagi muncul di daftar.

  1. Buka halaman detail dokumen
  2. Klik tombol merah "Hapus"
  3. Konfirmasi penghapusan pada dialog yang muncul

Audit Mutu Internal (AMI)

Panduan mengelola siklus lengkap audit mutu internal: dari perencanaan hingga penutupan.

6.1 Daftar Sesi Audit

Buka menu Audit Internal di sidebar. Halaman ini menampilkan seluruh sesi audit dalam bentuk kartu dengan informasi:

  • 📋 Nama SesiJudul sesi audit, contoh: "AMI Prodi S1 Keperawatan 2024"
  • 🏢 Unit AuditUnit atau prodi yang diaudit
  • 👤 AuditorNama auditor yang ditugaskan
  • 📅 PeriodeTanggal mulai dan selesai audit
  • 🏷️ StatusBadge berwarna: RENCANA (kuning), BERLANGSUNG (biru), SELESAI (hijau)

6.2 Membuat Sesi Audit Baru

⚠️ Hak Akses Hanya ADMIN dan AUDITOR yang dapat membuat sesi audit.
  1. Klik tombol "Buat Sesi Audit" di halaman daftar audit
  2. Isi formulir dengan data berikut:
Field Wajib Keterangan
Nama Sesi Audit Ya Nama deskriptif. Contoh: "AMI Prodi S1 Keperawatan 2024"
Unit / Prodi yang Diaudit Ya Pilih dari: S1 Keperawatan, D3 Kebidanan, S1 Farmasi, D3 Keperawatan, Profesi Ners, Institusi, LPPM, Perpustakaan
Tanggal Mulai Ya Tanggal rencana pelaksanaan audit dimulai
Tanggal Selesai Ya Tanggal rencana audit selesai
Auditor Ya Pilih auditor dari daftar pengguna dengan role AUDITOR atau ADMIN
  1. Klik "Buat Sesi" — Anda akan diarahkan ke halaman detail sesi audit yang baru dibuat.

6.3 Alur Status Audit

Setiap sesi audit melalui 3 tahap status:

📋 RENCANA
→
🔄 BERLANGSUNG
→
✅ SELESAI
Status Aksi yang Tersedia Keterangan
RENCANA Klik "Mulai Audit" Sesi baru dibuat, belum dimulai. Klik tombol untuk memulai proses audit.
BERLANGSUNG "Tambah Temuan" dan "Selesaikan Sesi" Audit sedang berjalan. Auditor dapat mencatat temuan. Tombol selesaikan aktif jika semua temuan sudah CLOSED.
SELESAI Hanya lihat (read-only) Sesi audit sudah ditutup. Tidak dapat menambah atau mengubah temuan.

6.4 Mencatat Temuan

Pada sesi audit yang berstatus BERLANGSUNG:

  1. Klik tombol "Tambah Temuan"
  2. Isi formulir temuan:
Field Wajib Keterangan
Deskripsi Temuan Ya Uraian detail tentang temuan audit yang ditemukan
Kategori Ya Pilih: Observasi, Ketidaksesuaian Minor, atau Major
Standar BAN-PT Tidak Kode standar BAN-PT yang terkait, contoh: C.4.2
Deadline Tindak Lanjut Tidak Tenggat waktu untuk menyelesaikan tindak lanjut
  1. Klik "Tambah" — Temuan akan muncul di daftar dengan status OPEN

6.5 Tindak Lanjut & Verifikasi Temuan

Setiap temuan memiliki alur penyelesaian:

🔴 OPEN
→
🟡 PROSES
→
🟢 CLOSED
  1. Pada kartu temuan, klik "Tindak Lanjut & Verifikasi" (untuk Admin/Auditor) atau "Isi Tindak Lanjut" (untuk Dosen)
  2. Isi kolom Tindak Lanjut dengan rencana atau hasil perbaikan
  3. Admin/Auditor dapat mengubah status verifikasi: OPEN → PROSES → CLOSED
  4. Klik "Simpan"
✅ Menyelesaikan Sesi Audit Sesi audit hanya dapat diselesaikan jika semua temuan sudah berstatus CLOSED. Tombol "Selesaikan Sesi" akan aktif secara otomatis.

Field Wajib Keterangan

Untuk melihat visualisasi data Monev:

  1. Di halaman Monev, klik link "Lihat Laporan" atau navigasi ke /monev/laporan
  2. Halaman laporan menampilkan:
    • Filter Periode — Pilih periode dari dropdown
    • Grafik Batang (Bar Chart) — Perbandingan visual Target vs Capaian per indikator
    • Legenda Warna — Hijau (tercapai), Kuning (mendekati), Merah (belum tercapai)
    • Tabel Rincian — Detail lengkap semua indikator beserta persentase dan status

7.5 Export Data CSV

  1. Di halaman Laporan Monev, klik tombol "Export CSV"
  2. File CSV akan otomatis terunduh dengan nama laporan-monev-[periode].csv
  3. File ini bisa dibuka di Microsoft Excel atau Google Sheets untuk analisis lebih lanjut
ℹ️ Data yang Ter-export File CSV berisi kolom: Indikator, Standar, Target, Capaian, Persen, dan Status.

FAQ & Troubleshooting

Pertanyaan umum dan solusi untuk masalah yang sering ditemui.

Saya tidak bisa login, muncul pesan "Email atau password salah"

Solusi:
1. Pastikan email dan password diketik dengan benar (perhatikan huruf besar/kecil)
2. Pastikan Anda menggunakan email yang didaftarkan oleh Admin
3. Jika masih gagal, hubungi Administrator untuk mereset password

Saya tidak melihat tombol Upload di halaman Dokumen

Solusi:
Tombol Upload hanya tersedia untuk role Admin, Auditor, dan Dosen. Jika Anda login sebagai Pimpinan, Anda hanya memiliki akses baca. Hubungi Admin jika role Anda perlu diubah.

Menu "Manajemen Pengguna" tidak muncul di sidebar

Solusi:
Menu ini hanya tampil untuk pengguna dengan role Admin. Ini adalah batasan hak akses yang disengaja untuk keamanan sistem.

Tombol "Selesaikan Sesi" tidak bisa diklik (disabled)

Solusi:
Sesi audit hanya dapat diselesaikan jika seluruh temuan sudah berstatus CLOSED. Pastikan semua temuan telah diverifikasi dan ditutup terlebih dahulu.

Bagaimana cara melihat laporan Monev dalam bentuk grafik?

Solusi:
Buka halaman Monitoring & Evaluasi, lalu klik link "Lihat Laporan". Halaman laporan menampilkan grafik batang dan tabel rincian. Anda juga bisa export data ke CSV.

Saya ingin mengubah email akun saya

Solusi:
Email tidak bisa diubah secara mandiri. Hubungi Administrator untuk membuat akun baru dengan email yang diinginkan.

Format file apa saja yang bisa diupload?

Jawaban:
Sistem mendukung file dengan format: PDF, DOC/DOCX (Word), XLS/XLSX (Excel), dan PPT/PPTX (PowerPoint). Ukuran maksimal sekitar 10MB per file.

Apa yang terjadi jika dokumen dihapus?

Jawaban:
Penghapusan dokumen bersifat soft delete. Dokumen tidak benar-benar hilang dari database, namun ditandai sebagai "dihapus" dan tidak muncul di daftar. Hanya Admin yang dapat melakukan penghapusan.
